1. What is the Hypertext Transfer Protocol (HTTP)?

    2. Hypertext Transfer Protocol (HTTP) is an application-layer and stateless protocol for transmitting hypermedia documents, such as HTML which was specially designed for communication between web browsers and web servers.

  2. Why is HTTP called a stateless protocol?

    3. HTTP is called a stateless protocol because the server does not keep any data between two requests i.e. every request are executed independently.

  3. Describe the three phases of establishing a session using client-server protocols such as HTTP.

    4. In client-server protocols, like HTTP, sessions consist of three phases:

    1. The client establishes a TCP connection (or the appropriate connection if the transport layer is not TCP).

      2. Client is the one who usually initiate connection in transport layer of TCP/IP. With TCP the default port, for an HTTP server on a computer, is port 80. Other ports can also be used, like 8000 or 8080. The URL of a page to fetch contains both the domain name, and the port number, though the latter can be omitted if it is 80.

    2. The client sends its request, and waits for the answer.

      3. A client request consists of text directives, separated by CRLF (carriage return, followed by line feed), divided into three blocks:

      First step done by the client in initiating a http connection is establishing a TCP connection with the http server.
      Subsequent lines gives information to server about which data is suitable.
      Final block contains additional block of data used by POST method.

    3. The server processes the request, sending back its answer, providing a status code and appropriate data.

      4. On getting the request from the client, the server responds, by revealing some information about itself, and metadata about the data asked by the client, along with the data. Then server sends a status code of 200 which means the request has succeeded to the user. Some of other status code are 301 which means URI of requested resource has changed, 404 for requested resource not found.

  4. Describe the parameters on a get and post request.

    5. GET

    When the GET request method is used, the client sends the server certain GET parameters through the requested URL. These parameters are pairs of names and their corresponding values, so-called name-value pairs. These are added to the URL with a “?” sign and tell the server which resources are meant. The name and value are always separated using a “=” sign.

    The syntax is as follows: http://www.domain.com/index.html*?name1=value1

    Multiple parameters, as well as entire lists, can be transmitted to the server. Here, the various parameters are separated using a “&” sign.
    http://www.domain.com/index.html*?name1=value1&name2=value2
    If the request is syntactically correct, the server sends a response.

    POST

    Post method is similar to get method mostly. But the only difference between them is that data sent through POST method will not visible in the URL.

  5. What are the HTTP response status code and What do the HTTP status codes, 200, and 404 indicate?

    6. HTTP response status codes are speicif codes that indicates whether a specific HTTP request has been successfully completed.
    They are grouped in five classes: informational responses, successful responses, redirects, client errors, and servers errors.
    Status Code 200 : The request has been completed successfully.
    Status Code 404 : The server cannot find the requested resource.

  6. What are HTTP cookies? Give an example.

    7. An HTTP cookie (web cookie, browser cookie) is a small piece of data that a server sends to the user's web browser. The browser may store it and send it back with the next request to the same server.
    Facebook uses cookies to verify a user's account and determine when s/he's logged in so facebook can make it easier for them to access the Facebook products and show them the appropriate experience and features.

  7. Why are cookies a security issue?

    8. Cookies are seen as security issues because hackers can exploit user's data and experience using them.

  8. Describe session hijacking and XSS

    9. Session hijacking

    In session hijacking, an attacker impels the user to use the attacker’s or another’s session ID. This can be done by using the cookie’s browser directive path, hence the user pretends to be someone else. Using this method, an attacker can urge the user to log in as the attacker on various application levels.

    XSS

    In order to carry out a cross-site scripting exploit, an attacker has to place the exploit in a cookie. Then the exploit vector will fetch the payload from the cookie and the exploitation is carried out. This type of attack will become difficult if the cookie has already been set; in this case, the attacker has to control the first cookie in the cookie string and only then can the attack be carried out.

  9. Describe Cross-site request forgery(CSRF).

    10. A browser sends a cookie in response to a request, regardless of where the request came from. This is where the actual problem with cookies comes in. When a website receives a request, it cannot distinguish whether the action is initiated by the user or not. It looks for the cookie and, if the cookie is available, it deliberately performs the action as if the user initiated it.
    Lets consider that a user “Rahul” browses through a legitimate website “www.xyz.com” and has a valid cookie on his hard disk. Meanwhile, an attacker, “Drokp,” embeds a link to perform some exchange action of “www.xyz.com” in an image and posts it on a site known as “www.hellos.com.” When the user Rahul visits “www.hellos.com,” the webpage loads the image and in turns gives a exchange request to “www.xyz.com.” When the web server receives the request, it looks for the cookie. It then finds Rahul’s cookie, interprets this as a valid request, and performs the exchange action.

  10. What are Cross-site HTTP requests (CORS) and what requests use CORS?

    11. (CORS) is a mechanism that uses additional HTTP headers to tell a browser to let a web application running at one origin (domain) have permission to access selected resources from a server at a different origin. A web application executes a cross-origin HTTP request when it requests a resource that has a different origin (domain, protocol, and port) than its own origin.

    Following requests use CORS:
    1. Invocations of the XMLHttpRequest or Fetch APIs in a cross-site manner
    2. Web Fonts
    3. WebGL textures.
    4. Images/video frames drawn to a canvas using drawImage().
